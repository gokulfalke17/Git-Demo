#Code


Q1: What is Git?

Git is a distributed version control system used to track file changes and collaborate.

Q2: What is commit?

A commit is a snapshot of your code at a specific time.

Q3: What is difference between commit and push?

Commit saves changes locally, push uploads to GitHub.

Q4: What is a branch?

A separate line of development.

Q5: What is merge conflict?

When two branches modify the same line of a file.

Q6: What is pull request?

A request to merge your branch into main after review.

Q7: What is fork vs clone?

Fork → copy repo into your GitHub
Clone → copy repo into your system

Q8: What is .gitignore?

File to ignore unnecessary files from repo.

Q9: What is stash?

Temporary storage for unfinished work.

Q10: What is "origin"?

Default name of remote GitHub repository.
